# Spec: 思考ログ（logモード）

## 背景 / 目的
ユーザーのモヤモヤ/課題を対話で深掘りし、言語化と気づきを促す。
終了後にタスク化（`#タスク整理`）へつなげる。

## スコープ
- **やること**
  - `#整理開始` でlogモード開始（`#ログ開始` はalias）
  - ユーザー発話ごとにDeepSeekで分析し、返信を3分割で返す
  - `#整理終了` で終了（`#ログ終了` はalias）
- **やらないこと**
  - logモード終了だけで自動タスク生成（タスク化は `#タスク整理` で明示実行）

## フロー（Given/When/Then）
- Given: 日報/ステータス等の別モードではない
- When: `#整理開始` を送る
- Then: 開始メッセージと、終了コマンド（`#整理終了`）が案内される

- Given: logモード中
- When: 任意の文章を送る
- Then: DeepSeek分析に基づき、返信が「共感/整理/核心質問」の3メッセージに分割される

- Given: logモード中
- When: `#整理終了` を送る
- Then: 終了メッセージが返り、次の `#タスク整理` が案内される

## 長文化ガード / 失敗時の導線（Given/When/Then）
- Given: logモード中
- When: ユーザー発話が増えて思考ログが長文化する
- Then: DeepSeekへ渡す入力は「先頭抜粋 + 直近抜粋」に圧縮され、対話が継続できる（全文は `sessions` に残る）

- Given: logモード中
- When: ユーザーとのやり取りが3往復くらい続く
- Then: 返信カードに「🧾 ここまでで整理終了」ボタンが出て、ユーザーが任意で区切れる（自動終了はしない）

- Given: logモード中
- When: DeepSeekが失敗する（タイムアウト/過負荷/例外など）
- Then: Webhook処理は落とさず、ユーザーに「いったん `#整理終了` で保存して区切る」導線（ワンタップ）を提示する

## 例外 / エラーハンドリング（未決定含む）
- DeepSeek失敗時の詳細（リトライ方針、保存のみ部分成功の範囲など）は `docs/bmad/gaps.md` で協議する
