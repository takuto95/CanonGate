# Spec: LINEセッション / コマンドルーティング

## 背景 / 目的
LINE上の入力は「モード（思考ログ/日報/ステータス）」と「コマンド」により解釈される。
モード中でも確実に動く“優先コマンド”や、セッション残留時の挙動を協議可能にする。

## モード（概念）
- **log（思考ログ）**
- **daily（日報）**
- **status（ステータス）**

## コマンド（例）
SSoT: `docs/bmad/spec-current.md` の各章に記載のコマンド群。

- **ヘルプ**: `/help`, `?`, `#ヘルプ` など
- **リセット**: `#リセット`, `reset`
- **設定**: `#設定 キャラクター ...`, `#設定 トーン ...`, `#設定 名前 ...`
- **タスク系（横断）**: `status <taskId>`, `split <taskId>`, `retry <taskId>`
- **ゴール系（横断）**: `#ゴール一覧`, `#ゴール進捗`, `#ゴール完了 <...>`
- **モード開始/終了**: `#整理開始/#整理終了`, `#日報開始/#日報終了`, `#ステータス` など

## 優先順位（ルーティング順）
以下は **spec-current記載の方針**をまとめたもの（詳細はSSoT参照）。

1. ヘルプ
2. リセット
3. パターンマッチの優先コマンド（設定/タスク系/ゴール系）
4. ステータスモード中の番号選択処理
5. キーワードコマンド（開始/終了など）
6. 日報/思考ログモード判定と通常入力

## セッション残留（未決定）
end無しセッションが残ると他モードがブロックされ得る。
復旧方針は `docs/bmad/gaps.md` の「セッション残留時の復旧」で協議する。

## 受入条件（Given/When/Then）
- Given: どのモード中でも
- When: `#設定` や `status t_xxx` 等の優先コマンドを送る
- Then: 「モードの入力待ち」ではなく、優先コマンドが実行される
