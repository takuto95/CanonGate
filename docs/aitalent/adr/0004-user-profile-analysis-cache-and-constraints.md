# ADR 0004: ユーザープロフィール（ロール）/ 分析サマリキャッシュ / 制約推定と上書き

- **Status**: Accepted
- **Date**: 2026-01-09
- **Deciders**: User（プロダクト方針）/ Agent（仕様整理）
- **Related**: `docs/bmad/spec-current.md`, `docs/bmad/gaps.md`, `docs/bmad/specs/04-thought-log-mode.md`, `docs/bmad/specs/05-task-summary-and-goal-auto-creation.md`

## Context
このプロダクトの価値は「AIとの対話で思考を言語化し、記録として蓄積し、本人の特性/傾向理解を元に“やるべきタスク”を提示・管理する」ことにある。

現状は:
- 対話の全文は `sessions` にイベントとして残る
- `logs` は主に日報サマリや `#タスク整理` 時に作られやすく、思考ログが“独立した資産”として蓄積されづらい
- パーソナライズ設定（`user_settings`）は、エージェントの話し方（キャラクター/トーン）中心で、本人のロール/制約を扱わない

また、制約（時間/環境/役割など）は「ユーザー入力に依存すると手間」だが、推定が外れた時に修正できないとUXが破綻する。

## Decision
以下を採用する。

1) **思考ログはタスク化と独立して蓄積する**
- `#整理終了` のタイミングで、思考ログの `logs` レコードを必ず1件作成する（タスク化は別）。

2) **タスクとログの紐付けを強化する**
- タスク生成時は `tasks.sourceLogId` に元ログの `logId` を保存する（追跡可能にする）。

3) **`logs` に種別と出典セッションIDを持たせる**
- `logs.type` と `logs.sourceSessionId` を追加し、`sessions`（全文）への参照を可能にする。

4) **`user_settings` に “分析サマリ（キャッシュ）” を持たせる（ユーザー編集不可）**
- エージェントは返信時にこのサマリを参照し、`sessions` を毎回読まずに“今のその人”を反映できるようにする。

5) **制約は推定を基本にしつつ、ユーザーが上書きできる**
- デフォルトは推定（ユーザー手間を最小化）。
- 推定が困難/不正確な場合に備えて、ユーザーが制約を変更できる導線を提供する。
 - **最低語彙（最小セット）**: 時間 / 環境（デバイス） / エネルギー
 - **TTL**: 上書きは「今日だけ」（日付が変わったら推定に戻る）

6) **ロールは「エージェントの話し方」ではなく「本人の立場/状況（プロフィール）」として扱う**
- 例: 仕事（管理職/リーダー等）、プライベート（独身/夫等）。
- これはキャラクターロール（`characterRole`）とは別概念で、ユーザーが明示できる情報とする。

## Options Considered
- Option A: `sessions`（全文）を分析の主データにする
  - 重い/増え続ける/過去がノイズになりやすい
- Option B: `logs`（要約/構造化）を分析の主データにする（今回採用）
  - 軽量・検索/集計向き。必要な時だけ `sourceSessionId` から全文参照が可能。
- Option C: `user_profile` を別シートとして新設して分析/制約/ロールを分離
  - 将来は有力だが、現状は `user_settings` 拡張で素早く進める（単一ユーザー運用のため）。

## Consequences
- **Positive**:
  - 思考ログが“独立資産”として増える（タスク化しなくても蓄積される）
  - 日常処理は `logs` と `user_settings.analysisSummary` を参照でき、軽量になる
  - `tasks.sourceLogId` で因果が追える（どのログから生まれたタスクか）
  - 推定制約が外れてもユーザーが修正できる
- **Negative / Risks**:
  - Sheetsスキーマ変更が必要（`logs` / `user_settings` の列追加）
  - 推定制約の誤りがUXに影響するため、上書き/失効/表示の設計が必要
- **Operational notes**:
  - スキーマ変更は `repair:sheets` 手順と互換を保つ（列名追加、既存列は保持）
  - 追加列は基本的に後方互換（空でも動作）にする
